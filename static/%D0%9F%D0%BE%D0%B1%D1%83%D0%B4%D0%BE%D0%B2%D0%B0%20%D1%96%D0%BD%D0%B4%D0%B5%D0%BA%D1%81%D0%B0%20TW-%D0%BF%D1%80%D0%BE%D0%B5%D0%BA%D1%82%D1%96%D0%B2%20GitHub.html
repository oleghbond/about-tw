Побудова індекса TW-проектів GitHub

29-4-2019
Різне

ПОСТАНОВКА ЗАВДАННЯ

Зібрати інформацію про всі проекти Тіддлівікі в обліковці GitHub і побудувати єдиний (індексний) файл представлення всіх проектів.

ПОСЛІДОВНІСТЬ КРОКІВ

  1. Скласти список всіх TW-проектів в обліковці GitHub. У нашому випадку це github/protw. З цього списку треба виключити організаційний (базовий) проект. Про базовий сайт обліковки на GitHub докладніше тут. Стисло кажучи, веб-адреси базового і решти проектів формуються наступним чином:
    Тип проектуСервісGitHub Pages
    БазовийGitHubgithub.com/<user_name>/<user_name>.github.io
    БазовийGitHub Pages<user_name>.github.io
    ЗвичайнийGitHubgithub.com/<user_name>/<proj_name>
    ЗвичайнийGitHub Pages<user_name>.github.io/<proj_name>

    У нашому випадку, якщо взяти <user_name> = protw і <proj_name> = tw5, то веб-адреси окремих проектів мають такі значення:

    Тип проектуСервісGitHub Pages
    БазовийGitHubgithub.com/protw/protw.github.io
    БазовийGitHub Pagesprotw.github.io
    ЗвичайнийGitHubgithub.com/protw/tw5
    ЗвичайнийGitHub Pagesprotw.github.io/tw5
  2. Для кожного TW-проекту в обліковці знайти файл index.html. Наприклад, у проекті tw5ghs обліковки protw (лінк) файл index.html розташований у субтеці docs проекту, а саме tw5ghs/docs/index.html (лінк).
  3. Екстрагувати зі знайденого файлу index.html два тідлера у div-форматі:
    • стандартний опис проекту $:/boa/desc/record і
    • стандартне лого проекту $:/boa/desc/image.
  4. Ковертувати два тідлери з div-формату у tid-формат.
    • Для цього спочатку кожний параметр перезаписати таким чином, наприклад:
      • з div-формату: created="20190416131023055"
      • у tid-формат: created: 20190416131023055 + Enter
    • Потім видобути вміст тідлера з div-формату між тегами <div><pre>вміст тідлера</pre></div> і помістити його у tid-формат після всіх параметрів + порожній рядок (Enter) (див. тут нижче Стандартне лого проекту у tid-форматі)
  5. У tid-форматі стандартного опису проекту і у стандартному лого проекту значення поля title змінити на значення з поля pub-title. У нашому прикладі для проекту TW5GHS це буде:
    • значення з div-формату: title="$:/boa/desc/record" та title="$:/boa/desc/image"
    • змінити на значення у tid-форматі: title: TW5GHS та title: TW5GHS.img, відповідно.
  6. Зберегти у tid-форматі стандартний опис проекту і стандартне лого проекту як файли у tid-форматі з іменами, що відповідають значенням поля title. У нашому прикладі це буде: TW5GHS та TW5GHS.img, відповідно.

    Місце збереження стандартних опису і лого проекту у базовому проекті - github.com/<user_name>/<user_name>.github.io/wiki/tiddlers/. у нашому випадку - у базовому проекті protw/protw.github.io у субтеці protw.github.io/wiki/tiddlers/ (лінк)

  7. На цьому порядок дій щодо одного проекту завершено, після чого треба повернутися до п. 3 вище доти, доки список проектів не буде вичерпано.
  8. Наостанок, залишається вирішити задачку з облаштування запуску процедури. В ідеалі треба знайти в API GitHub процедуру-тригер, щоб визначала подію зміни у конкретному проекті і передавала управління нашій процедурі з параметрами цього проекту.

СТАНДАРТНИЙ ОПИС ПРОЕКТУ $:/boa/desc/record

У середині файлу index.html стандартний опис проекту знаходиться у тег-дужках div зі значенням параметру title, що дорівнює $:/boa/desc/record

    title="$:/boa/desc/record" 
    tags="Портфоліо" 

    pub-author-date="2019" 
    pub-authors="Olegh Bondarenko" 
    pub-by="protw" 
    pub-date="20190416"

З урахуванням наявного сервіса веб-публікації ~GitHub Pages, що працює безпосередньо з вашого сховища ~GitHub, процес веб-публікації спрощується до роботи з текстовим редактором - ви просто редагуєте вікі, зберігаєте її і ваші зміни одразу стають реальністю у веб-просторі."
