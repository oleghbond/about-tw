29-1-2017
Встановлення ТВ

Від технічних можливостей хостинга залежить, який саме варіант ТВ можна розмістити на ньому:

  1. статичний HTML
  2. HTML+JS
  3. Node.js
  4. TW5 на Github pages

Нижче у таблиці надані апробовані варіанти хостингу:

ХостингВаріант ТВПримітки
Google Drive + HTML Viewer extension1(-) Нема доменного імені, незручний запуск, (+) є контроль доступу
Безкоштовний хостинг на базі DropBox updog.co2(-) повністю відкритий, (+) домен
На базі локально встановленого Node.js, синхронізація між користувачами з допомогою git (VCS) через платний сервер bitbucket.org3(-) відносно складне встановлення і підтримка оновлень документу, нема зовнішньої точки доступу (+) контроль версій і авторства, генерування статичних HTML
Безкоштовний хостинг TW5 на Github pages (детальна інструкція тут)4(-) досить нетрадиційний, але все колись було дуже нетрадиційним, важкуватий для розуміння запуск, відсутні вбудовані WYSIWYG інструменти, авторизація це окреме питання, (+) 1 тідлер - 1 файл, багатокористувацька відкрита вікі з відстежуванням версій, створення іншої вікі робиться через створення гілки з перейменуванням

Окремим, але нерозривним питанням завжди постає контроль доступу до ресурсу. Сам ТВ не надає засобів контролю цих питань. Але для конкретних хостів можна знайти відкриті інструменти автентифікації. Так, зокрема, для GitHub Pages це:

  1. Jekyll Auth
  2. Securing Site That Runs on Github Pages With JSON Backend In Private Repository

З обговоренням питань автентифікації для GitHub Pages можна ознайомитись на stackoverflow.
